<h1 align="center">osTicket - Ticket Lifecycle: Intake Through Resolution</h1>

<p>
This project demonstrates the lifecycle of a support ticket from intake through resolution using the open-source help desk ticketing system <strong>osTicket</strong>. The lab includes ticket creation, assignment, troubleshooting, communication, and final ticket resolution.
</p>

---

<h2>Project Summary</h2>

<p>
This tutorial walks through the complete lifecycle of a support ticket within osTicket. The project demonstrates how tickets are created by users, assigned to support agents, worked through by IT staff, and ultimately resolved and closed within the help desk environment.
</p>

<h3>Languages Used</h3>

- PowerShell

<h3>Environments Used</h3>

- Microsoft Azure (Virtual Machines)
- Windows 10 (21H2)
- Windows 11

<h3>Technologies / Applications Used</h3>

- osTicket
- Microsoft Azure
- Remote Desktop
- Internet Information Services (IIS)

---

<h2>Ticket Lifecycle Stages</h2>

- Ticket Intake
- Ticket Assignment
- Agent Communication
- Troubleshooting and Resolution
- Ticket Closure

---

<h3>Step 2: Ticket Assignment and Communication</h3>

<p>
After submission, the ticket is assigned to the appropriate department or support agent. Agents communicate with users through the ticket thread to gather additional information, provide updates, and document troubleshooting progress.
</p>

<br />

<h3>Step 3: Working the Issue</h3>

<p>
The assigned support agent investigates and troubleshoots the issue using the information provided in the ticket. Troubleshooting steps, escalations, and internal notes are documented within the osTicket platform to maintain accurate records.
</p>

<h3>Step 4: Resolution and Ticket Closure</h3>
